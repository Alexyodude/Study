SPI (Serial Peripheral Interface) is a **synchronous, full-duplex** bus commonly used to talk to sensors, displays, SD cards, and flash memory. It is faster than I2C but uses more wires.

## Bus Signals

| Signal | Direction (Master) | Purpose |
|--------|-------------------|---------|
| SCK | Output | Clock generated by master |
| MOSI | Output | Master Out, Slave In (data to slave) |
| MISO | Input | Master In, Slave Out (data from slave) |
| CS/NSS | Output | Chip Select -- active LOW to select a slave |

```
Master              Slave
  SCK  ----------->  SCK
  MOSI ----------->  MOSI
  MISO <-----------  MISO
  CS   ----------->  CS (active low)
```

Multiple slaves share SCK, MOSI, and MISO. Each slave has its own CS line -- only the selected slave responds.

## CPOL and CPHA: The Four SPI Modes

SPI has [four modes defined by two settings](https://fastbitlab.com/spi-cpol-cpha-discussion/):

- **CPOL** (Clock Polarity): idle state of SCK
- **CPHA** (Clock Phase): which edge samples data

| Mode | CPOL | CPHA | Clock Idle | Data Sampled On |
|------|------|------|------------|-----------------|
| 0 | 0 | 0 | LOW | Rising edge (1st edge) |
| 1 | 0 | 1 | LOW | Falling edge (2nd edge) |
| 2 | 1 | 0 | HIGH | Falling edge (1st edge) |
| 3 | 1 | 1 | HIGH | Rising edge (2nd edge) |

**Mode 0** is the most common (used by most sensors). Check your slave device's datasheet to confirm which mode it expects.

## Full-Duplex Operation

SPI is full-duplex: **every clock cycle shifts one bit out on MOSI and one bit in on MISO simultaneously**. Even when you only want to read, you must send something (usually 0x00 or 0xFF). Even when you only want to write, the slave sends something back (usually ignored).

```
Master TX:  [Command] [0x00] [0x00]    <-- sends command, then dummy bytes
Master RX:  [junk]    [data1] [data2]  <-- receives data during dummy bytes
```

## Key Registers (STM32)

| Register | Purpose |
|----------|---------|
| SPI_CR1 | Master/slave, baud rate prescaler, CPOL, CPHA, data frame size, SPI enable |
| SPI_CR2 | DMA enable, interrupt enable, frame format |
| SPI_SR | Status: TXE (TX empty), RXNE (RX not empty), BSY (busy) |
| SPI_DR | Data register -- write to send, read to receive |

### Important CR1 Bits

| Bit | Name | Purpose |
|-----|------|---------|
| 0 | CPHA | Clock phase |
| 1 | CPOL | Clock polarity |
| 2 | MSTR | 1 = master mode |
| 3:5 | BR[2:0] | Baud rate prescaler: f_PCLK / 2, 4, 8, ..., 256 |
| 6 | SPE | SPI enable |
| 7 | LSBFIRST | 0 = MSB first (default), 1 = LSB first |
| 8 | SSI | Internal slave select (when SSM=1) |
| 9 | SSM | Software slave management |
| 11 | DFF | Data frame format: 0 = 8-bit, 1 = 16-bit |

## Master Configuration Step-by-Step

<!-- tabs -->
```c
// 1. Enable clocks
RCC->APB2ENR |= RCC_APB2ENR_SPI1EN;
RCC->AHB1ENR |= RCC_AHB1ENR_GPIOAEN;

// 2. Configure GPIO pins for SPI1 (AF5 on STM32F4)
//    PA5 = SCK, PA6 = MISO, PA7 = MOSI
GPIOA->MODER |= (0x2 << (5*2)) | (0x2 << (6*2)) | (0x2 << (7*2));
GPIOA->AFR[0] |= (5 << (5*4)) | (5 << (6*4)) | (5 << (7*4));

// 3. Configure SPI1 as master
SPI1->CR1 = 0;                        // reset
SPI1->CR1 |= SPI_CR1_MSTR;            // master mode
SPI1->CR1 |= (0x3 << SPI_CR1_BR_Pos); // baud = f_PCLK/16
SPI1->CR1 &= ~SPI_CR1_CPOL;           // CPOL=0 (Mode 0)
SPI1->CR1 &= ~SPI_CR1_CPHA;           // CPHA=0
SPI1->CR1 |= SPI_CR1_SSM | SPI_CR1_SSI; // software CS management
SPI1->CR1 &= ~SPI_CR1_DFF;            // 8-bit data frame

// 4. Enable SPI
SPI1->CR1 |= SPI_CR1_SPE;
```

```rust
use core::ptr::{read_volatile, write_volatile};

const RCC_APB2ENR: *mut u32 = (0x4002_3800 + 0x44) as *mut u32;
const RCC_AHB1ENR: *mut u32 = (0x4002_3800 + 0x30) as *mut u32;
const GPIOA_MODER: *mut u32 = 0x4002_0000 as *mut u32;
const GPIOA_AFRL: *mut u32 = (0x4002_0000 + 0x20) as *mut u32;
const SPI1_CR1: *mut u32 = 0x4001_3000 as *mut u32;

unsafe {
    // 1. Enable clocks
    write_volatile(RCC_APB2ENR, read_volatile(RCC_APB2ENR) | (1 << 12)); // SPI1EN
    write_volatile(RCC_AHB1ENR, read_volatile(RCC_AHB1ENR) | (1 << 0)); // GPIOAEN

    // 2. Configure GPIO pins for SPI1 (AF5): PA5=SCK, PA6=MISO, PA7=MOSI
    let moder = read_volatile(GPIOA_MODER);
    write_volatile(GPIOA_MODER, moder | (0x2 << 10) | (0x2 << 12) | (0x2 << 14));
    let afrl = read_volatile(GPIOA_AFRL);
    write_volatile(GPIOA_AFRL, afrl | (5 << 20) | (5 << 24) | (5 << 28));

    // 3. Configure SPI1 as master
    write_volatile(SPI1_CR1, 0);  // reset
    let mut cr1: u32 = 0;
    cr1 |= 1 << 2;             // MSTR: master mode
    cr1 |= 0x3 << 3;           // BR: f_PCLK/16
    // CPOL=0, CPHA=0 (Mode 0) — already 0
    cr1 |= (1 << 9) | (1 << 8); // SSM + SSI: software CS
    // DFF=0: 8-bit — already 0
    write_volatile(SPI1_CR1, cr1);

    // 4. Enable SPI
    write_volatile(SPI1_CR1, read_volatile(SPI1_CR1) | (1 << 6)); // SPE
}
```

## SPI Transmit/Receive Function

<!-- tabs -->
```c
uint8_t spi_transfer(uint8_t data) {
    while (!(SPI1->SR & SPI_SR_TXE)) { }   // wait for TX buffer empty
    SPI1->DR = data;                         // send byte
    while (!(SPI1->SR & SPI_SR_RXNE)) { }  // wait for RX byte
    return SPI1->DR;                         // read received byte
}
```

```rust
use core::ptr::{read_volatile, write_volatile};

const SPI1_SR: *const u32 = (0x4001_3000 + 0x08) as *const u32;
const SPI1_DR: *mut u32 = (0x4001_3000 + 0x0C) as *mut u32;

unsafe fn spi_transfer(data: u8) -> u8 {
    while read_volatile(SPI1_SR) & (1 << 1) == 0 {}  // wait TXE
    write_volatile(SPI1_DR, data as u32);              // send byte
    while read_volatile(SPI1_SR) & (1 << 0) == 0 {}  // wait RXNE
    read_volatile(SPI1_DR as *const u32) as u8         // read received byte
}
```

## Example: Reading WHO_AM_I from an Accelerometer

Most SPI sensors have a WHO_AM_I register (read-only ID). For an LIS3DSH accelerometer, WHO_AM_I is at address 0x0F and should return 0x3F.

SPI sensor convention: bit 7 of the address byte = 1 for read, 0 for write.

<!-- tabs -->
```c
// CS pin = PA4 (manual GPIO control)
#define CS_LOW()  GPIOA->BSRR = (1 << (4 + 16))
#define CS_HIGH() GPIOA->BSRR = (1 << 4)

// Configure PA4 as output for CS
GPIOA->MODER |= (0x1 << (4*2));
CS_HIGH();  // deselect

uint8_t read_who_am_i(void) {
    CS_LOW();                          // select slave
    spi_transfer(0x0F | 0x80);        // address + read bit
    uint8_t id = spi_transfer(0x00);  // clock out response
    CS_HIGH();                         // deselect slave
    return id;                         // should be 0x3F
}
```

```rust
use core::ptr::{read_volatile, write_volatile};

const GPIOA_MODER: *mut u32 = 0x4002_0000 as *mut u32;
const GPIOA_BSRR: *mut u32 = (0x4002_0000 + 0x18) as *mut u32;

#[inline(always)]
unsafe fn cs_low() { write_volatile(GPIOA_BSRR, 1 << (4 + 16)); }
#[inline(always)]
unsafe fn cs_high() { write_volatile(GPIOA_BSRR, 1 << 4); }

unsafe {
    // Configure PA4 as output for CS
    write_volatile(GPIOA_MODER, read_volatile(GPIOA_MODER) | (0x1 << (4 * 2)));
    cs_high();  // deselect
}

unsafe fn read_who_am_i() -> u8 {
    cs_low();                              // select slave
    spi_transfer(0x0F | 0x80);            // address + read bit
    let id = spi_transfer(0x00);          // clock out response
    cs_high();                             // deselect slave
    id                                     // should be 0x3F
}
```

## Software vs Hardware CS

**Software CS (recommended for beginners):**
- Configure CS pin as regular GPIO output
- Assert/deassert manually with BSRR
- Full control over timing
- SSM and SSI bits set in CR1

**Hardware CS (NSS pin):**
- SPI peripheral controls the CS pin automatically
- Less flexible -- some slaves need CS held between bytes
- Useful for simple single-slave setups

Most real-world code uses software CS because it gives precise control over when CS toggles, which matters for multi-byte transactions.
